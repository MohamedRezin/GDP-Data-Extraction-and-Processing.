# Coffee Shop Relational Database Design & Normalization

## Overview

This project demonstrates the design and implementation of a normalized relational database for a multi-location coffee shop chain. The objective was to consolidate operational data from multiple business systems into a structured database that supports efficient transaction processing and business reporting.

The project simulates a real-world database engineering scenario involving schema design, normalization, relationship mapping, and reporting object creation.

## Project Scenario

A coffee shop chain plans to expand nationally and requires a centralized database system to manage operational and sales data.

As a Data Engineer, the task is to design a relational database that:

- Stores sales transaction data
- Tracks purchased products and quantities
- Maintains product and category information
- Manages staff and store location records
- Supports reporting through views and materialized views

## Objectives

- Identify entities and attributes from business requirements
- Design an Entity Relationship Diagram (ERD)
- Define primary and foreign key relationships
- Normalize the database structure
- Implement the schema in PostgreSQL
- Create reporting views and materialized views
- Maintain referential integrity across tables
